how this works (entropy, brute force, policy)

Pool entropy

Every character in your password is assumed to be drawn from a pool: lowercase (26), uppercase (26), digits (10), symbols (~32). Entropy in bits is length × log2(pool size). This is the theoretical maximum uncertainty an attacker faces if they know only the character set you used and nothing else.

Why "effective" entropy is lower

Real attackers don't brute-force blindly — they try leaked password lists, dictionary words, keyboard walks (qwerty), and common substitutions (@ for a, 0 for o) before pure brute force. This tool subtracts a penalty for each detected pattern to approximate that.

Crack time

Time = 2^(effective entropy) ÷ 2 ÷ guesses-per-second, assuming an offline attacker using a fast, poorly-chosen hash on modern GPU hardware (~10 billion guesses/second). A slow hash like bcrypt/argon2 could make the same password take thousands of times longer to crack.

Leaked-password check

Checks against an embedded list of common breached passwords (RockYou, "worst passwords" lists) — a small demo-scale sample rather than the full multi-million-entry corpus.
